# OneLedger 项目执行总则

> 适用范围：产品、设计、Android、数据、测试、Git 与发布

## 1. 产品承诺

OneLedger 是本地优先、快速、可信的个人记账工具。核心体验是：3 秒内开始记账，10 秒内完成一笔；打开首页即可理解本月余额；离线状态下核心功能完整可用。

设计原则：

1. **金额正确优先**：涉及金额、余额、预算、转账的修改必须有测试。
2. **本地优先**：不因网络或登录阻断记账；隐私功能默认保守。
3. **一眼读懂**：先显示结论，再显示分析；避免把统计术语交给用户解释。
4. **紧凑但可触达**：减少空白，不压缩 48dp 触控区域和必要层级。
5. **原创而一致**：参考功能，不复刻 iCost 的构图、图标、文案、色彩组合或动效路径。
6. **克制的反馈**：动效解释状态变化，不装饰等待时间。

## 2. 决策记录

会影响两个以上页面、数据库结构或发布计划的决定，必须写进对应规范或 ADR。记录至少包含：背景、方案、取舍、影响范围、决定日期。未经记录的临时决定不得成为长期约束。

## 3. 一次迭代的标准流程

### 需求就绪（Definition of Ready）

- 用户问题能用一句话说明。
- 明确包含项与不包含项。
- 列出默认、加载、空、错误、禁用、成功状态。
- 说明金额、时间、账户和数据库是否受影响。
- 给出可验证的验收条件，不使用“优化一下”“更高级”等模糊表述。

### 开发完成（Definition of Done）

- 主流程、取消流程和失败流程均可用。
- 深色/浅色、常见字号和窄屏不截断关键内容。
- 触控、语义标签、对比度及减少动态效果已检查。
- 单元测试、Lint、构建和相关截图测试通过。
- 文档、PR 描述和用户可见文案已更新。
- 没有明文密钥、真实财务数据、调试入口或无说明 TODO。

## 4. 变更控制

| 变更级别 | 示例 | 必须动作 |
| --- | --- | --- |
| L1 局部 | 文案、间距、小图标 | 自测 + 截图对比 |
| L2 功能 | 新页面、记账字段、交互路径 | 需求卡 + 测试 + PR 评审 |
| L3 高风险 | 数据库迁移、余额算法、备份恢复 | 设计记录 + 迁移/回滚测试 + 发布说明 |

任何 L3 变更不得和大面积 UI 重构放在同一个 PR 中。

## 5. 向项目负责人汇报

每次交付使用固定结构，文字清晰、先讲结果：

```text
完成：实现了什么，用户现在能做什么。
变化：涉及哪些页面、数据或交互。
验证：运行了哪些检查，结果如何。
注意：已知限制、风险或下一步。
文件：给出可点击的关键文件。
```

避免只说“已优化”“已处理”“应该没问题”；必须说明优化对象和可验证结果。
