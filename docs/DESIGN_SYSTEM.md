# OneLedger 视觉设计系统

## 1. 品牌语言

OneLedger 使用原创的“票据轨道”语言：票据切口表达记录，蓝绿轨道表达资金流，脉冲点表达交易发生，双轨日历同时展示收入与支出。参考产品只用于理解功能，不复制其页面比例、组件组合或视觉素材。

关键词：**可信、紧凑、清晰、克制、有流动感**。

## 2. 设计令牌

企业级设计系统应把颜色、尺寸、圆角、阴影和字体变成令牌，而不是在页面中散落常量。这与 [腾讯 TDesign 的 Design Token 方法](https://tdesign.tencent.com/starter/docs/vue/design-token)一致。

### 颜色

| 语义 | 浅/通用值 | 深色值或说明 |
| --- | --- | --- |
| 品牌主色 | `#145CC4` | `#75A4FF` |
| 品牌辅色 | `#10B7B4` | 同值，检查对比度 |
| 支出/危险 | `#D84C45` | `#FF6B5F` |
| 收入/正向 | `#168A63` | `#57D3A2` |
| 存钱/提醒 | `#9A6500` | `#FFC247` |
| 背景 | `#F5F7FB` | `#0B0D10` |
| 表面 | `#FFFFFF` | `#121820` |
| 高表面 | `#EEF1F7` | `#19212B` |
| 边框 | `#DDE2EA` | `#273442` |
| 主文字 | `#171A20` | `#F4F6FA` |
| 次文字 | `#697180` | `#8E96A3` |

颜色不单独承担含义：支出/收入同时使用负号/正号、标签或图标；超支需要“已超支”文字。

### 间距、圆角与尺寸

- 4dp 为微间距，8dp 为基础网格；常用间距为 8/12/16/24/32dp。
- 页面水平边距 16dp；同组元素 8–12dp，区块之间 20–24dp。
- 圆角：8dp 小控件、12dp 列表项、18dp 卡片、22dp Sheet、28dp 强调容器。
- 内容可更紧凑，但所有可交互区域至少 48×48dp；Android 官方无障碍指南同样建议 48dp 触控目标：[Accessibility](https://developer.android.com/design/ui/mobile/guides/foundations/accessibility)。
- 底部导航保持单行四项；系统导航区和手势区不得被内容覆盖。

### 排版

沿用系统无衬线字体，数字启用等宽数字特性。当前 Compose 基线：34/40 display、30/36 headline、22/28 section、19/24 title、16/24 body、14/20 secondary。金额允许比标题大一级，但同屏最多一个最大金额。

规则：金额和单位不拆行；中文正文不使用全大写英文；次要信息最低 12sp；放大字体后允许卡片增高，不裁切内容。

## 3. 组件原则

- **摘要卡**：一个结论、最多两个辅助指标、一个明确下钻入口。
- **账单行**：分类图标、标题/时间、金额/账户形成固定两列；支出保留负号。
- **主按钮**：每个页面最多一个；动词标签优于“确定”。
- **输入**：标签常驻，不用 placeholder 代替字段名；金额输入即时格式化但不改变用户光标意图。
- **Bottom Sheet**：适合快速记账和局部设置；复杂、长列表任务进入完整页面。
- **图表**：先给文字结论和精确数值，再给图形；图例不依赖颜色识别。
- **空状态**：一句解释 + 一个行动，不使用无关插画占满首屏。

组件必须覆盖 default、pressed、focused、selected、disabled、loading、error 状态。页面不得直接写品牌色十六进制，应从 `MaterialTheme` 或 OneLedger token 读取。

## 4. 原创性检查

提交 UI 前回答：

1. 页面是否使用 OneLedger 的品牌页眉、轨道或脉冲，而不是参考图的结构？
2. 信息分组、卡片比例、主色、图标和动效中是否至少三项明显不同？
3. 文案是否由 OneLedger 的任务语言重新编写？
4. 是否没有复制商标、头像、专有插画、截图素材和独特图标？

任一答案为“否”，该设计不能进入开发。

## 5. 无障碍与适配

- 正文与背景对比度目标至少 4.5:1，大文字至少 3:1。
- 图标按钮必须提供 `contentDescription`；装饰图标明确设为无语义。
- 支持 TalkBack 合理朗读，例如“本月可用预算九百二十四元五角”。
- 不用手势作为唯一入口；滑动切月必须同时有左右按钮。
- 支持系统深浅主题、字体缩放、横屏/大屏安全布局和“减少动态效果”。
- 删除、清空、覆盖备份等不可逆操作必须二次确认并说明影响。
