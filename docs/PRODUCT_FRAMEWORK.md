# OneLedger 产品与整体设计框架

## 1. 定位与边界

目标用户是希望快速记录、控制预算、理解资产，但不愿先注册或上传数据的个人用户。

v0.1 聚焦账本、资产、存钱、统计、预算与收支日历。账号、云同步、OCR、AI 自动入账、家庭协作和投资交易暂不进入 MVP。未来 AI/OCR 只能生成待确认草稿，不能静默修改账本。

## 2. 信息架构

```text
OneLedger
├─ 账本
│  ├─ 月预算摘要 → 预算详情 → 设置总/分类预算
│  ├─ 按日账单 → 账单详情/编辑
│  ├─ 收支日历 → 选中日期明细
│  └─ 快速记账 Sheet
├─ 资产
│  ├─ 净资产摘要
│  ├─ 账户列表
│  └─ 账户详情/新增账户
├─ 存钱
│  ├─ 方法入口
│  └─ 计划列表/详情
└─ 统计
   ├─ 周/月/年/自定义范围
   ├─ 收支与结余
   └─ 分类排行/趋势
```

四个底部目的地保持稳定。设置、导入导出和关于从页面菜单进入，不增加第五个主 Tab。

## 3. 核心任务

| 任务 | 最短路径 | 成功结果 |
| --- | --- | --- |
| 记一笔 | 任一主页 `+` → 类型/金额/分类 → 保存 | 列表、余额、预算和统计同步更新 |
| 修正账单 | 账单行 → 修改字段 → 保存修改 | 原账单更新，所有聚合同步重算 |
| 删除账单 | 账单行 → 删除 → 确认 | 账单软删除，并可从提示中撤销 |
| 看剩余预算 | 账本首页预算卡 | 显示总额、已用、可用/超支 |
| 调整预算 | 预算卡 → 预算详情 → 编辑 | 新预算写入 Room，并立即重算进度 |
| 看某日收支 | 收支日历 → 日期 | 同时看到支出、收入与当日明细 |
| 看净资产 | 资产页 | 区分总资产、总负债与净资产 |

## 4. 页面模板

每个页面按“识别—结论—行动—明细”组织：品牌/标题区用于识别；首屏摘要给结论；主按钮承载高频行动；列表展示可下钻证据。不得为了“高级感”让主金额、日期或保存按钮离开首屏。

所有页面必须定义以下状态：

- 默认：真实内容可读，首要行动明确。
- 加载：仅用于确有等待的操作；本地查询优先保持旧内容，不整页闪烁。
- 空：说明为什么为空，并提供唯一下一步。
- 错误：说明发生了什么、数据是否安全、如何恢复。
- 成功：就地更新为主；Snackbar 只补充可撤销或跨页面结果。
- 禁用：给出原因，不只降低透明度。

## 5. 数据与工程框架

```text
Compose UI → immutable UiState → ViewModel → Repository → Room DAO → SQLite
```

- 金额一律用最小货币单位 `Long`；展示时格式化。
- UI 不访问 DAO，不保存派生余额；余额和统计由交易聚合得出。
- 所有本地数据读取和写入以 Repository 的当前账本为边界；多账本 UI 上线前也不得回退到跨账本查询。
- 转账是同一业务事件，必须成对或以可证明一致的模型写入。
- 时间保存为时间戳，按设备时区展示；跨日边界必须测试。
- 数据库升级只能提供显式 Migration，禁止生产路径使用 destructive migration。
- 现阶段保持单 `app` 模块；只有团队边界、复用需求或构建瓶颈真实出现时再模块化。

## 6. 路线与优先级

优先级判断顺序：数据安全/错误 > 阻断记账 > 核心可理解性 > 性能与无障碍 > 新功能 > 装饰性视觉。

| 阶段 | 目标 | 出口条件 |
| --- | --- | --- |
| v0.1 | 可长期使用的本地账本 | CRUD、预算、资产、统计、导出稳定 |
| v0.2 | 完整个人财务 | 多账本管理、账户/分类 CRUD、转账、退款、周期账单、搜索 |
| v1.0 | 可公开发布 | 加密备份、无障碍、大屏、性能、隐私说明 |
| v2.0 | 可控智能输入 | OCR/自然语言先预览再入账，可完全关闭 |
