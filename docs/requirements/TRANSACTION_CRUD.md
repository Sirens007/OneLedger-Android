# 账单查看、编辑与删除

- 用户问题：已记录的账单无法打开、修正或删除，错误金额会持续影响余额、预算和统计。
- 目标指标：从账本首页或收支日历进入编辑不超过一次点击；保存或删除后相关聚合随 Room 数据流立即更新。
- 包含：账单行入口、复用记账 Sheet 编辑类型/金额/分类/账户/备注/时间、删除确认、删除后撤销、保存与删除失败反馈。
- 不包含：批量编辑、回收站、永久删除、报销与退款状态、多账本移动。
- 入口与退出：点击任一账单行进入；关闭或系统返回不保存；保存成功、删除成功后退出。
- 默认状态：显示现有账单全部可编辑字段，主操作为“保存修改”。
- 空状态：被其他数据更新移除的账单自动关闭编辑层，底层列表保持最新状态。
- 错误状态：保留用户已填写内容，并提示“账单未保存，请重试。已填写内容不会丢失。”；删除失败保持编辑层。
- 成功状态：就地刷新列表、余额、预算和统计；删除后 Snackbar 提供“撤销”。
- 禁用状态：金额无效、账户/分类缺失或转账账户相同时禁用保存，并保留当前输入。
- 数据影响：只更新现有 `transactions` 字段或写入/清除 `deletedAt`，数据库 schema 不变，不需要 Migration。
- 无障碍与减少动态效果：账单整行至少 48dp 可点击并提供合并语义；删除必须确认；Sheet 继续使用平台可中断过渡。

## 验收条件

1. Given 一笔支出，When 用户修改金额、分类或时间并保存，Then 原 ID 与创建时间不变，列表和所有聚合显示新值。
2. Given 一笔账单，When 用户确认删除，Then 账单从所有查询中消失，并出现可撤销提示。
3. Given 删除提示仍可操作，When 用户点击撤销，Then 同一账单恢复且不生成副本。
4. Given 保存或删除发生异常，When 操作结束，Then 编辑层和输入保持，用户看到可行动的错误文案。
5. Given 首页或收支日历中的账单，When 用户点击整行，Then 打开同一套编辑界面。
