# OneLedger 动效与用户文案规范

## A. 动效规范

### 1. 原则

动效只服务四件事：确认输入、说明来源与去向、维持空间连续性、帮助理解数据变化。用户操作后必须立即出现视觉反馈；动画可被打断；同类操作保持相同方向。频繁操作不使用夸张弹跳。

### 2. 动效令牌

| Token | 时长/参数 | 用途 |
| --- | --- | --- |
| `instant` | 0–80ms | 按下、高亮、开关反馈 |
| `fast` | 120–160ms | 图标、选中、数字局部变化 |
| `standard` | 180–240ms | Tab 容器、卡片展开、内容替换 |
| `sheet` | 260–320ms | Sheet/对话框进入与退出 |
| `springGentle` | damping 0.85，低位移 | 手势跟随后的归位 |

- 入场可用标准减速曲线；退场略快，避免阻挡下一步。
- 连续列表最多错开 20ms，总等待不得超过 120ms。
- 按钮按下立即缩放至 `0.97`，释放 160ms 回到 1；禁用按钮不缩放。
- 金额更新只动画变化数字或颜色，不让整个页面重新入场。

### 3. 场景规定

| 场景 | 规定 |
| --- | --- |
| 主 Tab | 内容淡化/局部替换；底栏选中容器 180–220ms 移动，不整屏横飞 |
| 切换月份 | 用户点击/滑动方向决定内容方向；标题和数据同步，支持中途反向 |
| 快速记账 | Sheet 从底部并与 `+` 的空间来源关联；背景轻微遮罩，不做背景缩放 |
| 保存成功 | 就地插入账单 + 轻触觉；需要撤销时显示 Snackbar |
| 预算展开 | 使用 `animateContentSize`，保持标题锚点稳定 |
| 日历选日 | 选中底色 120–160ms 变化；不放大单元格导致网格跳动 |
| 图表 | 首次进入可 240ms 渐进显示；筛选变化交叉过渡，不从零重播全部动画 |
| 错误 | 不抖动整屏；字段就地提示，必要时一次轻微横向反馈 |

### 4. Compose 实现选择

- 单值变化：`animate*AsState`。
- 多属性同步：`updateTransition`。
- 内容替换：`AnimatedContent`；出现/消失：`AnimatedVisibility`。
- 手势驱动和可中断动画：`Animatable`。
- 尺寸变化：优先 `animateContentSize`，并检查列表测量成本。

Compose 默认弹簧可中断；绘制阶段动画通常比反复触发布局更省成本。参考：[Compose animation quick guide](https://developer.android.com/develop/ui/compose/animation/quick-guide)、[Choose an animation API](https://developer.android.com/develop/ui/compose/animation/choose-api)。

### 5. 减少动态效果与性能

- 尊重系统动画缩放/减少动态效果；关闭位移和缩放，保留不超过 100ms 的淡化或即时状态切换。
- 不动画模糊半径、大面积阴影或整屏渐变。
- Lazy 列表中的动效以稳定 key 为前提，不因重组反复触发。
- 中端设备目标 60fps；动效不得推迟保存、返回或下一次点击。

## B. 用户可见文案

### 1. 语气

像可靠的财务助手：直接、克制、具体，不指责用户，不卖弄术语。先讲结果，再讲原因和行动。

### 2. 命名规则

- 按钮使用动作：`保存账单`、`设置预算`、`撤销`，避免泛化的`确定`。
- 页面标题使用名词：`预算详情`、`收支日历`、`账户`。
- 同一概念只用一个名称：统一使用“账单”“账户”“预算”“存钱计划”。
- `删除`用于可恢复单项；`清空`用于批量移除；`移除账户`必须说明账单是否保留。

### 3. 金额与时间

- 中文环境：`¥1,234.50`；支出 `-¥21.30`，收入 `+¥1,800.00`。
- 计算和存储不可用格式化后的字符串；小数位由币种决定。
- 同年日期：`7月19日`；跨年：`2027年1月2日`；相对时间仅用于近两日且旁边可查看准确时间。
- 预算状态统一为：`可用 ¥924.50`、`已用 ¥775.50`、`已超支 ¥40.98`。

### 4. 状态文案示例

| 场景 | 推荐文案 | 避免 |
| --- | --- | --- |
| 空账本 | `还没有账单，记下第一笔收支` | `暂无数据` |
| 保存成功 | `账单已保存` | `操作成功` |
| 保存失败 | `账单未保存，请重试。已填写内容不会丢失。` | `系统异常` |
| 无预算 | `本月还未设置预算` + `设置预算` | `预算未设置` |
| 超预算 | `本月已超支 ¥40.98` | `你花太多了` |
| 删除确认 | `删除这笔账单？删除后可在 5 秒内撤销。` | `确认删除？` |
| 加载 | `正在导出账单…` | `Loading…` |

### 5. 开发者向用户汇报

所有交付说明遵循“完成—变化—验证—注意”结构。技术名词首次出现时用普通话解释，例如“Room（手机本地数据库）”。不要向用户展示内部堆栈、数据库 ID 或无处理建议的错误码。
